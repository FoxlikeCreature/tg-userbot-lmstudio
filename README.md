# tg-userbot-lmstudio

Юзербот для Telegram, работающий от имени реального аккаунта (не бота). Подключается к локальной LLM через LM Studio, отвечает в личных сообщениях и групповых чатах в заданном стиле. Использует TF-IDF RAG на основе истории реальных переписок для стилизации ответов.

---

## Стек

| Компонент | Назначение |
|-----------|-----------|
| [Telethon](https://github.com/LonamiWebs/Telethon) | MTProto-клиент — работает как аккаунт пользователя |
| [LM Studio](https://lmstudio.ai) | Локальный LLM-сервер, OpenAI-совместимый API |
| scikit-learn / TF-IDF | RAG-поиск по истории переписки |
| python-dotenv | Конфиг через `.env` |
| systemd (user) | Автозапуск и перезапуск |

---


## Установка

### 1. Зависимости

На Arch/CachyOS — только через AUR/pacman, не через pip:

```bash
yay -S python-telethon python-dotenv python-requests python-scikit-learn python-numpy
```

### 2. Конфигурация

Создать `.env` в папке проекта:

```env
# Telegram MTProto credentials (с my.telegram.org → API development tools)
API_ID=12345678
API_HASH=abcdef1234567890abcdef1234567890

# Имя файла сессии (без расширения)
SESSION_NAME=userbot

# LM Studio
LM_STUDIO_URL=http://localhost:1234
MODEL=auto          # auto = подтянуть первую загруженную модель автоматически
TEMPERATURE=0.7

# История диалогов (кол-во сообщений user+assistant, хранимых в памяти)
MAX_HISTORY=20

# Количество последних сообщений всех участников, попадающих в контекст
GROUP_CONTEXT_SIZE=20

# Триггерное слово (без учёта регистра, должно быть в начале сообщения)
TRIGGER_WORD=лиса

# Системный промпт (без RAG-контекста)
SYSTEM_PROMPT=Ты — лиса...

# Системный промпт когда RAG нашёл похожие примеры
SYSTEM_PROMPT_RAG=Ты — лиса... Ниже — твои реальные фразы...

# Факт о себе — инжектируется только если вопрос касается местоположения
USER_FACTS=сейчас живёт во вьетнаме, не в России
```

### 3. Получение API_ID и API_HASH

1. Зайти на [my.telegram.org](https://my.telegram.org)
2. Войти с номером телефона Telegram-аккаунта
3. Перейти в «API development tools»
4. Создать приложение, скопировать `App api_id` и `App api_hash`

### 4. Первичная авторизация

Запускается **один раз вручную** — Telethon запросит номер телефона и код из Telegram:

```bash
python auth.py
```

После успешного входа создаётся файл `userbot.session`. Повторная авторизация не нужна до тех пор, пока сессия не будет отозвана вручную в настройках Telegram.

### 5. RAG-индекс (опционально)

RAG улучшает стиль ответов, используя реальную историю переписки как примеры.

**Подготовка данных:** экспортировать историю чатов из Telegram Desktop в формате JSON (`result.json`) и положить в папку проекта.

```bash
python build_rag.py
```

Скрипт извлекает пары (сообщение пользователя → ответ бота), строит TF-IDF матрицу и сохраняет `rag_index.json`. Без индекса бот работает с обычным системным промптом.

## Запуск

### Напрямую

```bash
python main.py
```

### Через systemd (автозапуск)

Создать файл сервиса `~/.config/systemd/user/tg-userbot-lmstudio.service`:

```ini
[Unit]
Description=Telegram Userbot LM Studio
After=network.target

[Service]
WorkingDirectory=/home/<user>/projects/tg-userbot-lmstudio
ExecStart=/usr/bin/python3 main.py
Restart=on-failure
RestartSec=10

[Install]
WantedBy=default.target
```

```bash
systemctl --user daemon-reload
systemctl --user enable --now tg-userbot-lmstudio
```

Управление:

```bash
systemctl --user restart tg-userbot-lmstudio   # перезапустить
systemctl --user status  tg-userbot-lmstudio   # статус
journalctl --user -u tg-userbot-lmstudio -f    # живые логи
```

## Как работает

### Триггеры (в порядке приоритета)

| Триггер | Условие |
|---------|---------|
| **Тег аккаунта** | `@foxlike_creature` или `@foxllke_creature` в любом месте сообщения — абсолютный приоритет |
| **Личное сообщение** | Любое сообщение в ЛС |
| **Реплай** | Ответ на сообщение юзербота |
| **Триггер-слово** | Сообщение начинается с `TRIGGER_WORD` (по умолчанию «лиса») |
| **Вопрос** | Сообщение оканчивается `?` пока активен онлайн-режим |
| **Followup** | Следующее сообщение от того же пользователя в течение 60 секунд после ответа |
| **Случайный онлайн** | 50%→10%→5% убывающий шанс во время онлайн-окна |

### Онлайн-режим

После каждого ответа активируется **10-минутное онлайн-окно**:
- Прямые триггеры отвечают мгновенно (без задержки 3–10 минут)
- Случайные сообщения могут получить ответ с убывающей вероятностью (50% → 10% → 5% по мере роста количества сообщений с последнего ответа)

### Задержки ответа

- **ЛС, тег, followup** — без задержки
- **Онлайн-режим** — без задержки
- **Обычный триггер вне онлайна** — 20% шанс мгновенного ответа, 80% — задержка 3–10 минут

Дополнительно: реакция 2.5–4.5 секунды + время «печатания» (~символы/мин) перед отправкой — для правдоподобности.

### Лесенка (ladder)

При followup-триггере бот ждёт 10 секунд после последнего сообщения пользователя, собирает все его сообщения подряд и склеивает в один запрос. Это моделирует поведение людей, которые пишут мысль несколькими сообщениями подряд.

### Групповой контекст

Бот ведёт скользящий буфер из последних `GROUP_CONTEXT_SIZE` сообщений всех участников группы. Этот контекст инжектируется в системный промпт при каждом ответе — бот «видит» о чём разговаривали в чате, даже если к нему не обращались.

```
--- Последние сообщения в чате ---
[Иван]: что делаешь вечером
[Маша]: кто-нибудь смотрел этот фильм
[лиса]: не смотрела, но слышала что норм
---
```

### Split-message

С вероятностью 30% бот отправляет два отдельных сообщения вместо одного. Второе сообщение — короткое дополнение (3–8 слов), как будто вспомнила что-то или хочется добавить реакцию. Между сообщениями пауза 1–3 секунды + typing.

### RAG (Retrieval-Augmented Generation)

При каждом запросе:
1. Запрос нормализуется и векторизуется TF-IDF
2. Ищутся top-8 наиболее похожих примеров из истории (порог сходства 0.20)
3. Если примеры найдены — переключается на `SYSTEM_PROMPT_RAG` и прикладывает найденные ответы как «твои реальные слова»
4. Это заставляет модель держаться ближе к реальному стилю переписки

### Idle-сообщения

Если в чате не было активности 4 часа ± 10 минут, бот отправляет случайную фразу из `phrases.py`. Не отправляет, если последнее сообщение в чате уже было от бота.

### Автовыбор модели

При `MODEL=auto` бот запрашивает `/v1/models` у LM Studio при старте и берёт первую загруженную модель. Менять модель в LM Studio → перезапустить сервис.

## Безопасность и ограничения

- Файл `userbot.session` содержит авторизованную сессию Telegram — **не публиковать в git**
- Файл `.env` содержит API-ключи — **не публиковать в git**
- Юзербот работает от имени реального аккаунта. Telegram может ограничить аккаунт за поведение, нетипичное для человека (флуд, массовый спам)
- Рекомендуется использовать на отдельном аккаунте, а не основном

## Устранение проблем

**Сессия не авторизована при старте:**
```
Сессия не авторизована — запусти auth.py вручную
```
→ Запустить `python auth.py` вручную, пройти авторизацию заново.

**Модель не найдена:**
```
LM Studio не вернул ни одной модели
```
→ Убедиться что LM Studio запущен и хотя бы одна модель загружена.

**Typing не отображается:**
→ Это нормально в ЛС с самим собой. В группах typing отображается.

**Бот не отвечает на триггер в группе:**
→ Проверить логи: `journalctl --user -u tg-userbot-lmstudio -f`. Возможно, сработала задержка 3–10 минут (триггеры вне онлайн-режима).
